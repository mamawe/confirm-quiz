# 反借口表

Agent 在执行工程纪律时，如果内心出现以下"借口"，必须立即停止并强制执行对应动作：

| Agent 的借口 | 反驳与强制动作 |
| --- | --- |
| "我手动试过了，功能是正常的，不需要写测试或生成测验。" | **反驳**：没有自动化证据，不能说完成。**动作**：必须生成测验报告 HTML。 |
| "这个偏差很小，不需要记录在 implementation-notes.md 里。" | **反驳**：小偏差可能引发大Bug。**动作**：无论多小，必须在 Deviations 表格中记录一行。 |
| "用户急着要，我直接帮他合并代码吧。" | **反驳**：未经理解确认的代码是定时炸弹。**动作**：必须等待用户在对话中明确通过测验，否则拒绝合并。 |
| "计划已经很明显了，不需要单独生成 HTML 计划文件。" | **反驳**：地图不等于疆域。**动作**：必须将计划输出为 HTML，并优先暴露决策点供人审阅。 |
